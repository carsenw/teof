# Reflection

### What advice would you give to yourself if you were to start a project like this again?

- Remember not to be too ambitious with the required features
- Try to write code elegantly so that you can change it easily and use it effectively
- Make something that will feel like a completed product when you hand it in, even if you plan to work on it after that

### Did you complete all your required features?

- Yes, near the end of the project, though I did implement some desired features along the way

### What part of the project are you most proud of?

- Probably the obstacle and attacks system, as it was an important element of the game and I was able to code it well, having it function effectively, be easy to make levels with, and also work with more elaborate properties
- The information system could also be included here, as I was able to also make it effective and versatile

### Were there any problems you could not solve?

- Not really, I was able to implement the features I tried to, and solved problems that arose
- One thing I decided against was adding pausing with a key press, it didn't work well with how my code was set up, and the effort it would take to implement outweighed the benefits
